## Weekly Schedule Template

### Weekday (Mon-Fri): 2-3 hours/day

```
Hour 1: Theory/Course Material
- Read course material
- Watch video tutorials
- Take notes

Hour 2-3: Hands-on Practice
- Lab machines
- Challenge boxes
- Tool practice
```

### Weekend (Sat-Sun): 5-6 hours/day

```
Morning Session (3-4 hours):
- Extended lab sessions
- Full machine walkthroughs
- Challenge exercises

Afternoon Session (2-3 hours):
- Write-ups and documentation
- Tool development
- Review and planning
```

## Tracking Progress

### Metrics to Track

1. **Machines Rooted**: Target 100+ before exam
2. **Write-ups Created**: Document everything
3. **Time per Machine**: Track improvement
4. **Vulnerabilities Found**: Build pattern recognition
5. **Tools Mastered**: Proficiency checklist

### Monthly Review Checklist

- [ ] Review goals vs actual progress
- [ ] Identify weak areas
- [ ] Adjust study plan if needed
- [ ] Update skill inventory
- [ ] Plan next month's focus

## Contingency Planning

### If Behind Schedule

- Reduce CTF participation
- Focus on core OSCP topics
- Extend timeline if needed (don't rush)
- Seek community help on Reddit/Discord

### If Ahead of Schedule

- Complete more challenging boxes
- Help others in the community
- Deepen AD knowledge
- Practice more report writing

### If Feeling Burnt Out

- Take a week break
- Do something non-technical
- Come back refreshed
- Adjust study hours temporarily

## Success Criteria

Before scheduling the exam, ensure:

- [ ] Rooted 50+ boxes (mix of easy, medium, hard)
- [ ] Completed all PWK lab machines
- [ ] Passed 3+ mock exams
